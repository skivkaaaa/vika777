Лабораторная работа № 2
Тема: «Структуры данных: мультисписок/очередь/дек/приоритетная очередь»

Цель работы заключалась в изучении и практическом применении линейных структур данных на языках программирования Python, C++ и Java. В ходе работы были рассмотрены мультисписки, очереди, деки и приоритетные очереди.

Мультисписок представляет собой структуру данных, содержащую другие списки в качестве элементов. Существует две разновидности: вложенные списки и двусвязные списки. Очередь работает по принципу FIFO (первый вошел - первый вышел), где добавление элементов происходит в конец, а извлечение - из начала. Дек является двунаправленной очередью, поддерживающей операции с обоих концов. Приоритетная очередь обрабатывает элементы согласно их важности, а не порядку поступления.

При сравнении реализации мультисписков было выявлено, что Python обеспечивает наибольшую простоту благодаря динамической типизации и встроенным спискам. C++ требует ручного управления памятью, но дает полный контроль над производительностью. Java занимает промежуточное положение, предлагая автоматическое управление памятью и готовый класс LinkedList.

Для работы с очередями и деками в Python доступны модули collections и queue, что значительно упрощает реализацию. В C++ используются std::queue и std::deque, обеспечивающие типобезопасность и эффективность. Java предоставляет интерфейсы Queue и Deque с различными реализациями, включая LinkedList и ArrayDeque.

Приоритетные очереди в Python реализуются через модули heapq и PriorityQueue, в C++ требуется перегрузка операторов для пользовательских типов, а в Java настройка осуществляется с помощью компараторов.

В заключение можно отметить, что каждый язык имеет свои преимущества: Python отличается простотой и скоростью разработки, C++ обеспечивает максимальную производительность, а Java предлагает сбалансированное решение с богатой стандартной библиотекой. Выбор языка зависит от конкретных требований проекта: Python подходит для прототипирования, C++ - для высокопроизводительных систем, Java - для кроссплатформенных приложений.
